# Local Data Safety

이 문서는 개인 데이터를 사용하는 로컬 설치의 DB 업그레이드, 백업·복원, 내보내기와
등록 문서 삭제 경계를 설명합니다. 모든 산출물은 Git에서 제외된 로컬 `data/` 영역에
저장되며 GitHub 업로드 대상으로 간주하지 않습니다.

## 스키마 업그레이드

앱은 DB를 일반 쓰기 연결로 열기 전에 역할과 스키마 버전을 읽기 전용으로 확인합니다.
요청 모드와 DB 역할이 다르거나 앱보다 새로운 스키마이면 즉시 중단합니다. 업그레이드가
필요한 기존 DB에는 다음 순서를 적용합니다.

1. 미반영 WAL 변경이 없는지 확인합니다.
2. DB 파일을 owner-only 백업 폴더에 바이트 단위로 복사합니다.
3. 백업의 무결성, 외래키, 역할과 스키마 버전을 확인합니다.
4. 모든 스키마 변경과 migration 이력 기록을 한 트랜잭션으로 처리하고, 각 버전의 이름과
   SHA-256 체크섬이 현재 릴리스 manifest와 정확히 같은지 확인합니다.
5. 완료된 DB의 무결성, 외래키, 역할과 버전을 다시 확인합니다.
6. 어느 단계든 실패하면 WAL sidecar를 제거하고 검증된 임시 파일의 원자적 rename으로
   변경 전 DB 파일·권한·시간을 복구합니다.

새 설치도 동일한 migration manifest를 기록하므로 운영 중 만들어진 DB와 신규 DB의
스키마 출처를 비교할 수 있습니다.

## 점검과 백업

`npm run doctor -- --mode=personal`은 Node 버전, 로컬 전용 host·port, 설정 완료 여부,
DB 권한·역할·무결성·스키마 버전과 migration 이름·체크섬을 검사합니다.

`npm run db:backup -- --mode=personal`은 원본과 SHA-256이 같은 owner-only SQLite 파일을
만듭니다. 열린 DB에 미반영 WAL 데이터가 있으면 오래된 main 파일만 복사하지 않도록
중단합니다.

## 복원

복원 명령은 기본적으로 후보 백업을 임시 파일에 복사해 무결성과 체크섬, DB 역할을
확인하는 dry-run만 수행합니다. 실제 교체에는 `--write`와 정확한 확인 문구
`RESTORE_LOCAL_DATABASE`가 함께 필요합니다. 대시보드와 다른 DB 클라이언트를 먼저
종료해야 하며 WAL·SHM 파일이 있으면 복원을 거부합니다. 교체 직전 현재 DB도 별도
안전 백업으로 보존하고, 설치 검증이 실패하면 이 안전 백업으로 되돌립니다. 검증된 임시
DB를 같은 디렉터리에서 원자적으로 rename해 설치하므로 기존 파일 위에 부분 복사된 DB가
노출되지 않습니다.

## 로컬 내보내기

`npm run db:export -- --mode=personal`은 모든 사용자 테이블을 owner-only JSON으로
내보냅니다. 이 파일에는 프로필, 공고, 지원 상태, 문서 메타데이터와 지원 문안이 포함될
수 있으므로 저장소에 추가하거나 메시지·이메일에 첨부하면 안 됩니다. 내보내기 폴더와
파일 형식은 `.gitignore`와 보안 스캐너의 차단 범위입니다.

## 등록 문서 보존과 삭제

등록 문서는 사용자가 직접 비활성화하더라도 자동 삭제되지 않습니다. 보존 기간 정리
명령의 기본 동작은 다음 조건을 모두 만족하는 항목의 목록만 출력하는 dry-run입니다.

- `source_documents.active = 0`
- 사용자가 지정한 일수보다 마지막 변경 시각이 오래됨
- 파일이 로컬 private data 경계 안의 일반 파일이며 심볼릭 링크가 아님

실제 삭제에는 `--write`와 정확한 확인 문구 `DELETE_EXPIRED_PRIVATE_DOCUMENTS`가 모두
필요합니다. 파일은 먼저 private 격리 폴더로 원자 이동하고 DB 행과 감사 이벤트를 한
트랜잭션으로 반영합니다. DB 단계가 실패하면 파일을 원래 위치로 되돌립니다. 마지막
파일 제거가 실패하면 원본 경로로 되돌리지 않고 owner-only 격리 상태와 감사 이벤트를
남깁니다.

## 범위 밖

- 개인 DB를 원격 저장소로 자동 백업하거나 동기화하지 않습니다.
- 대시보드 실행 중인 DB를 온라인 복원하지 않습니다.
- 활성 문서나 지원 패키지를 보존 기간만으로 자동 삭제하지 않습니다.
- 잡플래닛 회사 평점·리뷰를 수집·저장·분석하지 않습니다.
